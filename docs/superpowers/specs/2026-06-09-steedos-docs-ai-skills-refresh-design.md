# Steedos Docs AI Skills 文档翻新设计

Date: 2026-06-09

## 目标

为 Steedos 3.0 官网文档新增一个聚焦的 AI 辅助开发与 Steedos Skills 小节，主要面向使用 `create-steedos-app` 创建或维护 Steedos 项目的开发者。

同时新增一份很薄、工具中立的 `AGENTS.md`，让 Codex、Claude、GitHub Copilot 以及类似 AI 编程助手知道：当前仓库是 Docusaurus 文档站源码，不是 Steedos 平台运行时源码仓库。

## 已确认方案

采用 Developer 下的小型独立小节，而不是只在现有页面里补一句说明，也不是重写整个 Developer 文档。

这样可以让 AI 辅助开发在官网里有一个正式入口，同时让第一轮改动保持聚焦。更完整的 `skills/*/SKILL.md` 参考仍保留在 platform 仓库中，不整批搬运到官网文档站。

## 范围

本次包含：

- 新增一个 Developer 下的 AI 辅助开发小节。
- 在 `docs/` 下新增英文 current docs。
- 在 `i18n/zh-CN/docusaurus-plugin-content-docs/current/` 下新增对应中文 current docs。
- 在现有 Developer 入口页中增加一小段中英文导流。
- 在仓库根目录新增 `AGENTS.md`。

本次不包含：

- 不修改 `/Users/tujiajun/Documents/Github/steedos-platform-3`；该仓库只作为只读参考资料。
- 不修改 `versioned_docs/`，也不修改 `i18n/zh-CN/docusaurus-plugin-content-docs/version-2.7/` 等旧版本 i18n 文档。
- 不移动或重写现有 Developer、No-Code、Admin、Deploy、Platform 栏目。
- 不为 Codex、Claude、Copilot、Cursor 等工具编写未经验证的专属安装步骤。

新增对应中文 current docs 结构：

现有 `sidebars.js` 使用 autogenerated Developer 目录，应能自动识别新增小节，不需要额外手写 sidebar 配置。

## 页面内容

`01-overview.md` 说明 AI 辅助开发在 Steedos 中的位置：开发者描述业务需求，AI 协助生成或修改 metadata 与 package 文件，人工负责复核业务规则、权限模型与生产可发布性。

`02-steedos-skills.md` 说明 Steedos Skills 是给 AI 编程助手使用的 Steedos 领域知识库。页面应使用 platform 参考仓库中已有的命令，例如 `npx skills add steedos/steedos-platform`、`--all`、`--skill`、`--list`，并提供按任务选择 skill 的路由说明。

`03-ai-dev-testing.md` 面向官网读者说明如何在他们自己的 Steedos 项目中使用 AI 助手进行迭代开发与测试：先让 AI 产出计划，再在用户项目中实施 metadata/package 改动，启动 Steedos 项目，人工验证；复杂场景可选 Playwright MCP 浏览器自动化测试；最后把错误、截图或观察结果反馈给 AI 助手继续迭代。该页面描述的是用户项目的推荐流程，不是本次 docs 仓库实施阶段的验证流程。

中文页面应使用自然的中文文档表达，不做逐字机器翻译。

## AGENTS.md

在仓库根目录创建一份很薄的 `AGENTS.md`，包含：

- 项目事实：当前仓库是 `docs.steedos.com` 的 Docusaurus 源码。
- 参考边界：`steedos-platform-3` 只是只读参考资料。
- 文档目录：英文 current docs、中文 current docs、旧版本 docs 的位置。
- 验证命令：`yarn start`、`yarn start:cn`、`yarn build`、`yarn build:cn`；只有 OpenAPI 文件变化时才需要 OpenAPI 构建命令。
- AI 助手注意事项：保持改动小而清晰，current docs 新增/移动/重命名页面时中英文同步，不在 docs 仓库里使用 Steedos platform 运行时验证命令。

该文件应保持工具中立，让 Codex、Claude、GitHub Copilot 以及类似助手都能读懂。

## 分支与 Review 流程

所有改动都在 `codex/docs-ai-skills-refresh` 分支上进行。

第一个 commit 只包含本设计文档。用户 review 并确认后，再进入 Superpowers 的 implementation planning 步骤。正式文档与 `AGENTS.md` 的实际改动，只能在 implementation plan 获批后开始。

## 验证

设计文档 commit 阶段只验证文件内容和 git 状态。

后续正式实施阶段运行：

```bash
yarn build
yarn build:cn
```

除非 OpenAPI 文件发生变化，否则不需要运行 OpenAPI 构建。

## 完成标准与实施自检

正式实施完成后，执行者需要自查并在结果中报告：

- 英文和中文 current docs 中都存在新的 Developer AI 辅助开发页面。
- 新页面可通过 Developer sidebar 访问，并从 Developer 入口页获得链接。
- 发布到官网的文档内容只引用公开仓库或文档 URL，不引用本机绝对路径。
- `AGENTS.md` 薄、事实化、工具中立。
- platform 参考仓库和旧版本文档保持不变。
- `yarn build` 和 `yarn build:cn` 已执行并说明结果。
